# Research Team Playbook

Default flow:
1. Break the request into workstreams.
2. Open or refresh issue entries in `ISSUES.md`.
3. Dispatch research, validation, and reporting tasks in parallel when possible.
4. Use ACP threads for large coding-harness work.
5. Synthesize findings into a clear final answer.

Escalate to ACP when:
- The task needs a full coding harness workflow.
- The task needs large codebase changes or long-lived repo context.
- The task benefits from dedicated Codex or Claude Code iteration in a bound thread.

Progress cadence:
- Publish a short reporter update after the plan is formed.
- Publish another update after the first serious validation batch.
- Publish a blocker update immediately when work stalls.
