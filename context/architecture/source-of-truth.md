# OpenClaw Source-of-Truth Model

## Goal
Make this repo sufficient to rebuild the intended OpenClaw deployment on a fresh server without reading the current Oracle host by hand.

## Scope boundary
- This repo is the source of truth for our intended deployment state.
- OpenClaw product behavior is not defined by this repo.
- OpenClaw is an open-source upstream project that changes frequently, so generic behavior, config semantics, and feature support should be verified against `https://docs.openclaw.ai` and `https://github.com/openclaw/openclaw`.

## Authoritative tracked state
- `build/<host>/rootfs/`: the desired host file tree with exact host-style paths.
- `build/<host>/secrets.example.env`: the secret contract for the host-local `.secrets/<host>.env` file.
- `scripts/render-build-state.sh` and `scripts/apply-build-host.sh`: deterministic render/apply flow for fresh-host rebuilds.

## Local-only state
- `.secrets/<host>.env`: local-only secret values that are merged into the rendered build during apply.

## Rendered host outputs
These files are materialized locally into `.tmp/rendered-build/<host>/` and then uploaded:
- `~/.openclaw/openclaw.json`
- `~/.openclaw/.env`
- `~/.openclaw/acp-harness.env`
- `~/.claude/settings.json`
- `~/.config/systemd/user/openclaw-gateway.service.d/acp-harness.conf`

## Generated or runtime-only state
These files may exist on the host and are useful for audits/live captures, but they are not the source of truth for rebuilds:
- `~/.config/systemd/user/openclaw-gateway.service`: generated by `openclaw gateway install --force` and `openclaw doctor --repair`.
- `~/.openclaw/control-ui/*`: generated install assets.
- `~/.openclaw/tools/*`: downloaded browser/runtime tooling.
- `~/.openclaw/openclaw.json.bak*`: local safety backups only.
- `~/.openclaw/discord/thread-bindings.json`: live thread/session routing state.
- `~/.openclaw/logs/*`, `~/.openclaw/memory/*`, `~/.openclaw/subagents/*`: runtime history and caches.
- `~/.openclaw/identity/*`: host identity/runtime device state.
- `~/.openclaw/workspace*/.openclaw/workspace-state.json`: local workspace runtime state.
- `~/.openclaw/workspace*/.openclaw/*`: ad hoc runtime helper material inside agent workspaces, including sandbox-local GitHub CLI binaries/config, is operational state, not tracked deploy state.
- `~/.openclaw/exec-approvals.json`, `~/.openclaw/sandbox/containers.json`, `~/.openclaw/devices/*`: live operational state that may drift.

## Fresh-host rebuild workflow
1. Install Node 22+ on the target host.
2. Copy `build/<host>/secrets.example.env` to `.secrets/<host>.env` and fill the secrets locally.
3. Edit `build/<host>/rootfs/` until it matches the intended host state.
4. Render locally: `./scripts/render-build-state.sh --build-dir build/<host> --secrets-file .secrets/<host>.env`
5. Apply remotely: `./scripts/apply-build-host.sh --host <ssh-host> --secrets-file .secrets/<host>.env`
6. Optionally capture the live host into `.tmp/live/<host>/` for comparison.

## Coverage status
- As of 2026-03-11 UTC, the Oracle deployment's declarative settings are captured in `build/oracle.ylioo.com/rootfs/` and the local-only `.secrets/oracle.ylioo.com.env`.
- Runtime-only host state is intentionally excluded from the tracked deploy tree.

## Audit policy
- Every server interaction should be recorded in `operation-logs/`.
- `build/` is the declarative rebuild layer.
- `.tmp/live/` is the untracked redacted live-capture layer.
