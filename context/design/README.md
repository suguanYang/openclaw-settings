# Design Records

Use this directory for future-facing function design so proposal work does not get mixed into live-host notes.

## Layout
- `proposals/`: drafts for new functions, refactors, or workflow changes.
- `decisions/`: accepted design records after approval or implementation.

## Minimum sections
- Status
- Scope
- Motivation
- Approach
- Affected paths
- Rollout and verification
- Follow-up

## Naming
- Use `YYYY-MM-DD-short-name.md` for both proposals and decisions.
